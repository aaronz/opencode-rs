## 3. 测试执行通用规范
### 3.1 用例执行顺序（必须严格按顺序执行，禁止乱序）
1. TUI核心启动与基础命令测试（阻塞用例集）
2. TUI启动标志全量覆盖测试
3. TUI远程attach连接功能测试
4. TUI与会话管理联动功能测试
5. TUI与代理/模型/认证联动功能测试
6. TUI全局标志与环境变量适配测试
7. TUI异常场景与容错测试
8. 关联CLI命令与TUI兼容性测试

### 3.2 用例结果判定标准
| 结果标记 | 官方对齐判定规则 |
|----------|----------|
| PASS | 所有官方定义的预期行为100%满足，所有校验点全部命中，无报错、无fatal日志、进程行为符合预期 |
| FAIL | 任意一个校验点未命中，出现官方文档未定义的报错、崩溃、进程异常退出、行为与官方定义不符 |
| SKIP | 前置依赖用例执行失败，或环境不满足官方定义的执行条件，必须明确标注跳过原因 |

### 3.3 失败用例处理规则
1. 阻塞用例（核心启动类）执行FAIL，立即终止全量测试，输出测试报告与失败详情
2. 非阻塞用例执行FAIL，完整记录失败原因、执行日志、进程状态，继续执行后续用例
3. 单步骤超时，重试1次，再次超时直接标记为FAIL，记录超时信息

### 3.4 日志记录规范
每个用例执行完成后，必须记录以下信息：
- 用例ID、用例名称、所属模块、官方文档依据
- 执行开始/结束时间、耗时
- 执行输入：完整命令、参数、环境变量配置
- 全量stdout/stderr输出、进程PID、退出码、运行状态
- 每个校验点的命中情况
- 最终结果（PASS/FAIL/SKIP）
- 失败/跳过原因（若有）
