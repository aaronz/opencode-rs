# Module PRD: v2 (Session V2 Schema & Event System)

## 1. Module Overview

| Field | Value |
|-------|-------|
| **Module Name** | v2 |
| **Source Path** | `packages/opencode/src/v2/` |
| **Type** | Schema / Event System |
| **Rust Crate** | `opencode-session-v2` (or within `opencode-core`) |
| **Purpose** | Defines the next-generation session data model with a streaming event system, structured session entries, and an immutable state-stepping mechanism for real-time agent interaction rendering |

## 2. Functionality

The v2 module provides:

1. **Structured Event Types** (`session-event.ts`): A complete union of typed streaming events emitted during an agent session. Events are fine-grained (text delta, tool input delta, step started/ended, etc.) and carry timestamps and optional metadata.

2. **Session Entry Model** (`session-entry.ts`): Higher-level aggregated views of session history — `User`, `Synthetic`, `Assistant`, `Compaction` entries. An `Assistant` entry collects all streaming events within a step (text chunks, tool calls, retries) into a structured content list.

3. **State Stepper** (`session-entry-stepper.ts`): A pure, functional state machine that applies streaming events to session state. Uses an `Adapter` pattern to support in-memory stepping and arbitrary backends. Immutable state updates via `immer`-style produce.

4. **Session Service V2** (`session.ts`): A thin Effect service layer wrapping the v1 session service, with v2-typed inputs/outputs. Provides `create`, `prompt`, and `fromID` operations.

### Key Behaviors

- **One-directional event stream**: All session changes are modeled as events; the entry list is a derived view.
- **Tool lifecycle tracking**: Tool call state transitions: `pending → running → completed | error`.
- **Pending queue**: User prompts that arrive while an assistant is streaming are queued in `pending`, not `entries`.
- **Compaction**: Context compression events are represented as `Compaction` entries.
- **Retry tracking**: Failed LLM calls accumulate as `AssistantRetry` items inside the `Assistant` entry.

### session-event.ts

```typescript
// Core event union
export namespace SessionEvent {
  export const Event: Schema.Union<[Prompt, Synthetic, Step.Started, Step.Ended,
    Text.Started, Text.Delta, Text.Ended,
    Tool.Input.Started, Tool.Input.Delta, Tool.Input.Ended,
    Tool.Called, Tool.Success, Tool.Error,
    Reasoning.Started, Reasoning.Delta, Reasoning.Ended,
    Retried, Compacted]>

  export type Event  // = the union type
  export type Type   // = Event["type"] — discriminant string

  export const ID: Schema.BrandedString<"Session.Event.ID">

  // Event classes with .create() factory and Schema.Class fields:
  export class Prompt { text, files?, agents?, ...Base }
  export class Synthetic { text, ...Base }
  export namespace Step {
    export class Started { model: { id, providerID, variant? }, ...Base }
    export class Ended { reason, cost, tokens: { input, output, reasoning, cache }, ...Base }
  }
  export namespace Text {
    export class Started { ...Base }
    export class Delta { delta, ...Base }
    export class Ended { text, ...Base }
  }
  export namespace Reasoning {
    export class Started { ...Base }
    export class Delta { delta, ...Base }
    export class Ended { text, ...Base }
  }
  export namespace Tool {
    export namespace Input {
      export class Started { callID, name, ...Base }
      export class Delta { callID, delta, ...Base }
      export class Ended { callID, text, ...Base }
    }
    export class Called { callID, tool, input, provider: { executed, metadata? }, ...Base }
    export class Success { callID, title, output?, attachments?, provider, ...Base }
    export class Error { callID, error, provider, ...Base }
  }
  export class Retried { attempt, error: RetryError, ...Base }
  export class Compacted { auto, overflow?, ...Base }

  export class FileAttachment { uri, mime, name?, description?, source? }
  export class AgentAttachment { name, source? }
  export class Source { start, end, text }
  export class RetryError { message, statusCode?, isRetryable, responseHeaders?, responseBody?, metadata? }
}
```

### session-entry.ts

```typescript
export namespace SessionEntry {
  export type ID = SessionEvent.ID

  // Top-level entry union
  export const Entry: Schema.Union<[User, Synthetic, Assistant, Compaction]>
  export type Entry
  export type Type = Entry["type"]

  export class User { id, type:"user", text, files?, agents?, time:{created}, metadata? }
  export class Synthetic { id, type:"synthetic", text, time:{created}, metadata? }
  export class Assistant {
    id, type:"assistant", content: AssistantContent[],
    retries?: AssistantRetry[], cost?, tokens?, error?, time:{created, completed?}, metadata?
    static fromEvent(event: Step.Started): Assistant
  }
  export class Compaction { id, type:"compaction", auto, overflow?, time:{created}, metadata? }

  // Assistant content items
  export const AssistantContent: Schema.Union<[AssistantText, AssistantReasoning, AssistantTool]>
  export class AssistantText { type:"text", text }
  export class AssistantReasoning { type:"reasoning", text }
  export class AssistantTool {
    type:"tool", callID, name, state: ToolState,
    time: { created, ran?, completed?, pruned? }
  }

  // Tool state machine
  export const ToolState: Schema.Union<[ToolStatePending, ToolStateRunning, ToolStateCompleted, ToolStateError]>
  export class ToolStatePending { status:"pending", input:string }
  export class ToolStateRunning { status:"running", input:Record<string,unknown>, title?, metadata? }
  export class ToolStateCompleted { status:"completed", input, output, title, metadata, attachments? }
  export class ToolStateError { status:"error", input, error, metadata? }

  export class AssistantRetry { attempt, error: RetryError, time:{created} }
}
```

### session-entry-stepper.ts

```typescript
export type MemoryState = {
  entries: SessionEntry.Entry[]
  pending: SessionEntry.Entry[]
}

export interface Adapter<Result> {
  getCurrentAssistant(): SessionEntry.Assistant | undefined
  updateAssistant(assistant: SessionEntry.Assistant): void
  appendEntry(entry: SessionEntry.Entry): void
  appendPending(entry: SessionEntry.Entry): void
  finish(): Result
}

// Create an in-memory adapter
export function memory(state: MemoryState): Adapter<MemoryState>

// Apply one event to state via adapter (returns adapter.finish())
export function stepWith<Result>(adapter: Adapter<Result>, event: SessionEvent.Event): Result

// Apply one event immutably (pure function, immer-style)
export function step(old: MemoryState, event: SessionEvent.Event): MemoryState
```

## 6. Acceptance Criteria

- [ ] All 18 event types in `SessionEvent.Event` are representable as distinct Rust enum variants
- [ ] Event IDs are branded strings with ascending order property (sortable by time)
- [ ] `SessionEntry.Entry` union covers user, synthetic, assistant, compaction variants
- [ ] `AssistantTool` tracks the 4-state tool lifecycle: pending → running → completed | error
- [ ] `stepWith` pure function correctly transitions state for all 18 event types
- [ ] Prompt events during active assistant step are queued in `pending`, not appended to `entries`
- [ ] Step.Ended sets `completed` timestamp on the current assistant entry
- [ ] Retry events append to `assistant.retries` without creating a new entry
- [ ] Compaction events create a `Compaction` entry in the entry list
- [ ] `step()` function is pure (no mutation of input state)
- [ ] `memory()` adapter produces correct `MemoryState` after finish

## 7. Rust Implementation Guidance

### Crate
`opencode-core` (session_v2 module) or `opencode-session-v2`

### Key Crates
- `serde` / `serde_json` — serialization
- `chrono` — UTC timestamps
- `uuid` — event ID generation (use ascending ULIDs)
- `ulid` — ordered ID generation (preferred over uuid for "evt" prefix)
- `enum_dispatch` or hand-rolled match — event union dispatch

### Recommended Approach

```rust
// Event ID
#[derive(Debug, Clone, PartialEq, Eq, PartialOrd, Ord, Serialize, Deserialize)]
pub struct EventId(String);  // "evt_<ulid>"

impl EventId {
    pub fn create() -> Self {
        Self(format!("evt_{}", ulid::Ulid::new()))
    }
}

// Event union
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "type", rename_all = "kebab-case")]
pub enum SessionEvent {
    Prompt(PromptEvent),
    Synthetic(SyntheticEvent),
    #[serde(rename = "step.started")]
    StepStarted(StepStartedEvent),
    #[serde(rename = "step.ended")]
    StepEnded(StepEndedEvent),
    #[serde(rename = "text.started")]
    TextStarted(TextStartedEvent),
    #[serde(rename = "text.delta")]
    TextDelta(TextDeltaEvent),
    // ... etc
}

// Session entry union
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "type", rename_all = "lowercase")]
pub enum SessionEntry {
    User(UserEntry),
    Synthetic(SyntheticEntry),
    Assistant(AssistantEntry),
    Compaction(CompactionEntry),
}

// Tool state machine
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "status", rename_all = "lowercase")]
pub enum ToolState {
    Pending { input: String },
    Running { input: HashMap<String, serde_json::Value>, title: Option<String> },
    Completed { input: HashMap<String, serde_json::Value>, output: String, title: String, metadata: HashMap<String, serde_json::Value> },
    Error { input: HashMap<String, serde_json::Value>, error: String },
}

// State stepper
pub struct MemoryState {
    pub entries: Vec<SessionEntry>,
    pub pending: Vec<SessionEntry>,
}

pub fn step(old: &MemoryState, event: &SessionEvent) -> MemoryState {
    let mut state = old.clone();
    step_mut(&mut state, event);
    state
}
```

### Pattern Note
The TypeScript `immer` `produce` pattern maps to clone-and-mutate in Rust. Use `Clone` + `mut` local variable for the same semantics. The `Adapter` trait maps to a Rust trait for testability.

## 8. Test Design

No dedicated test files exist for the v2 module in the TypeScript repository (it's a schema/model layer still under development). However, the following test cases SHOULD be implemented based on the state machine logic:

### Unit Tests

```rust
#[cfg(test)]
mod tests {
    use super::*;

    // 1. step() is pure — does not mutate input
    #[test]
    fn step_does_not_mutate_input() {
        let initial = MemoryState { entries: vec![], pending: vec![] };
        let event = SessionEvent::Prompt(PromptEvent { text: "hello".to_string(), ..Default::default() });
        let _ = step(&initial, &event);
        assert!(initial.entries.is_empty());
    }

    // 2. Prompt event with no active assistant → added to entries
    #[test]
    fn prompt_added_to_entries_when_no_active_assistant() {
        let initial = MemoryState::default();
        let event = make_prompt_event("Hello");
        let next = step(&initial, &event);
        assert_eq!(next.entries.len(), 1);
        assert!(matches!(next.entries[0], SessionEntry::User(_)));
        assert!(next.pending.is_empty());
    }

    // 3. Prompt during active assistant → queued in pending
    #[test]
    fn prompt_queued_when_assistant_active() {
        let mut initial = MemoryState::default();
        // Add active (incomplete) assistant entry
        initial.entries.push(SessionEntry::Assistant(make_assistant(None)));
        let event = make_prompt_event("During");
        let next = step(&initial, &event);
        assert_eq!(next.entries.len(), 1); // assistant still there
        assert_eq!(next.pending.len(), 1); // prompt queued
    }

    // 4. StepStarted closes previous assistant and opens new one
    #[test]
    fn step_started_completes_previous_assistant() {
        let mut initial = MemoryState::default();
        initial.entries.push(SessionEntry::Assistant(make_assistant(None)));
        let event = make_step_started_event();
        let next = step(&initial, &event);
        // Previous assistant should have completed timestamp set
        if let SessionEntry::Assistant(a) = &next.entries[0] {
            assert!(a.time.completed.is_some());
        }
        // New assistant added
        assert_eq!(next.entries.len(), 2);
    }

    // 5. Tool state: pending → running → completed
    #[test]
    fn tool_state_transitions() {
        let state = apply_events(vec![
            make_step_started_event(),
            make_tool_input_started("call1", "bash"),
            make_tool_called("call1"),
            make_tool_success("call1", "output text"),
        ]);
        let assistant = get_assistant(&state.entries);
        let tool = get_tool(&assistant.content, "call1");
        assert!(matches!(tool.state, ToolState::Completed { .. }));
    }

    // 6. Tool state: running → error
    #[test]
    fn tool_error_transitions_to_error_state() {
        let state = apply_events(vec![
            make_step_started_event(),
            make_tool_input_started("call1", "bash"),
            make_tool_called("call1"),
            make_tool_error("call1", "permission denied"),
        ]);
        let assistant = get_assistant(&state.entries);
        let tool = get_tool(&assistant.content, "call1");
        assert!(matches!(tool.state, ToolState::Error { .. }));
    }

    // 7. Text delta accumulation
    #[test]
    fn text_deltas_accumulate_in_assistant_text() {
        let state = apply_events(vec![
            make_step_started_event(),
            make_text_started_event(),
            make_text_delta("Hello"),
            make_text_delta(" World"),
        ]);
        let assistant = get_assistant(&state.entries);
        let text = get_text_content(&assistant.content);
        assert_eq!(text, "Hello World");
    }

    // 8. Reasoning delta accumulation
    #[test]
    fn reasoning_deltas_accumulate() {
        let state = apply_events(vec![
            make_step_started_event(),
            make_reasoning_started_event(),
            make_reasoning_delta("Think"),
            make_reasoning_delta("ing"),
        ]);
        let assistant = get_assistant(&state.entries);
        let reasoning = get_reasoning_content(&assistant.content);
        assert_eq!(reasoning, "Thinking");
    }

    // 9. Retry events tracked in assistant.retries
    #[test]
    fn retry_events_appended_to_assistant_retries() {
        let state = apply_events(vec![
            make_step_started_event(),
            make_retried_event(1),
        ]);
        let assistant = get_assistant(&state.entries);
        assert_eq!(assistant.retries.as_ref().map(|r| r.len()), Some(1));
    }

    // 10. Compaction creates Compaction entry
    #[test]
    fn compacted_event_creates_compaction_entry() {
        let state = apply_events(vec![make_compacted_event(true)]);
        assert!(matches!(state.entries[0], SessionEntry::Compaction(_)));
    }

    // 11. StepEnded sets cost and tokens on assistant
    #[test]
    fn step_ended_sets_cost_and_tokens() {
        let state = apply_events(vec![
            make_step_started_event(),
            make_step_ended_event(0.05, 100, 200),
        ]);
        let assistant = get_assistant(&state.entries);
        assert_eq!(assistant.cost, Some(0.05));
        assert!(assistant.tokens.is_some());
    }

    // 12. Event IDs are sortable/ascending
    #[test]
    fn event_ids_are_ascending() {
        let id1 = EventId::create();
        std::thread::sleep(std::time::Duration::from_millis(1));
        let id2 = EventId::create();
        assert!(id1 < id2);
    }
}
```
