# Implementation Plan v14 — TUI 完整实现

**Version**: 14.0  
**Date**: 2026-04-07  
**Based**: spec_v14.md (FR-201 ~ FR-225)  
**Status**: 草稿

---

## 1. Executive Summary

基于 `TUI_PRD_Rust.md` (v2.4 Rust Edition) 和当前 `crates/tui/` 实现状态审计，本计划将 TUI 从半成品提升为生产可用的终端用户界面。

### 当前状态

| 维度 | 状态 | 备注 |
|------|------|------|
| 架构基础 | ✅ 已建立 | 15 个源文件/目录，模块化结构合理 |
| 核心交互 | ⚠️ 部分实现 | 事件循环/输入/文件引用/Shell 基础存在 |
| 斜杠命令 | ⚠️ 部分实现 | 框架存在，命令不完整 |
| 组件系统 | ⚠️ 部分实现 | 基础组件存在，缺消息气泡/代码块 |
| Markdown渲染 | ❌ 缺失 | 需集成 pulldown-cmark + syntect |
| 主题系统 | ⚠️ 部分实现 | 基础存在，缺多主题切换 |
| 状态栏 | ❌ 缺失 | 需新建 |
| CLI 参数 | ❌ 缺失 | 需新建 |
| 权限模式 | ❌ 缺失 | 需新建 |
| 依赖版本 | ⚠️ 需升级 | ratatui 0.28 → 0.30 |

### 实施策略

**分 6 个 Phase，P0 优先**：

| Phase | 主题 | FR 覆盖 | 预计工时 | 优先级 |
|-------|------|---------|----------|--------|
| Phase 0 | 基础清理与升级 | 基础设施 | 2d | P0 |
| Phase 1 | 核心交互完善 | FR-201~208, 212, 213, 220~222, 224 | 8d | P0 |
| Phase 2 | 布局与主题增强 | FR-209, 210, 211, 214 | 4d | P1 |
| Phase 3 | 流式输出与可视化 | FR-215, 216, 219 | 4d | P1 |
| Phase 4 | 高级功能 | FR-217, 218, 223 | 4d | P2 |
| Phase 5 | 性能优化与验收 | FR-225 | 2d | P2 |
| **总计** | - | **25 FR** | **24d** | - |
