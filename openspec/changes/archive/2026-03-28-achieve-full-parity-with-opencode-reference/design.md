## Context

当前Rust实现的opencode项目(rust-opencode-port)与TypeScript参考项目(opencode)之间存在功能差距。核心模块、CLI命令和LLM Providers已经实现，但存在以下问题：

1. **编译错误** - schema_validation.rs中的生命周期标注问题导致无法编译
2. **CLI路由未完成** - main.rs中子命令路由缺失
3. **测试基础设施问题** - 测试中使用临时值导致生命周期错误

## Goals / Non-Goals

**Goals:**
- 修复所有编译错误，使项目可以正常构建
- 完成CLI命令路由，使所有子命令可调用
- 修复测试基础设施问题
- 通过TypeScript项目的测试用例验证功能对等

**Non-Goals:**
- 不添加新的LLM Providers (已实现)
- 不重构核心架构
- 不实现新的功能模块

## Decisions

### 1. CLI命令路由架构

**决策**: 使用yargs风格的子命令模式

**理由**: 
- TypeScript项目使用类似模式
- 易于扩展和维护
- 与现有代码风格一致

**备选方案**:
- 使用clap - 需要重写现有实现
- 使用自定义路由 - 增加维护成本

### 2. 测试修复策略

**决策**: 使用静态空Vec替代临时vec![]

**理由**:
- 解决生命周期问题
- 最小化代码改动
- 保持测试逻辑不变

## Risks / Trade-offs

**[风险] CLI命令可能返回错误** → 通过实现完整的命令处理逻辑来缓解

**[风险] 测试运行时间可能较长** → 使用增量构建和并行测试来优化
